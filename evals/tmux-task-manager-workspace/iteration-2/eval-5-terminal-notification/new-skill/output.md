`sent a terminal notification` 表示这个后台任务在它所属的 task slot 里触发了终端铃声或终端通知。按照这个 skill，它本身**不是**成功、失败或结束的结论，而是一个“需要查看上下文”的信号。

这通常意味着几种可能：
- 程序主动发出了 bell/notification，用来提示有重要事件
- 任务可能遇到了需要关注的状态变化
- 有时也可能接近“等待输入”或输出了值得立即检查的信息

agent 的正确处理方式是：

1. **先把它当成需要检查的提醒，不要直接判定成功或失败**
   - 不能因为出现 notification 就说任务完成了
   - 也不能直接把它当成报错

2. **立即检查这个已有任务的上下文**
   - 优先检查触发通知的那个已存在 task slot
   - 如果通知里同时给了 `window id` 和 `task name`，应优先把 `window id` 当作精确定位依据
   - 在同一个共享的项目 task session 里查看该任务最近输出了什么

3. **判断通知原因**
   - 看看是不是正常提示信息
   - 看看是不是已经进入交互提示、卡在输入、或出现警告/错误
   - 如果任务实际上已经失败，再按 failed 的流程处理
   - 如果任务是在等输入，再按 waiting-for-input 的方式优先处理

4. **只在有依据时再决定下一步**
   - 若只是普通提示，通常不需要重启任务
   - 若发现阻塞、异常或非零退出，再解释原因并决定是否在**同一个 task name** 下重跑

简而言之：

> `sent a terminal notification` 的意思是“这个后台任务发出了需要关注的终端提醒”，agent 应该**检查该任务上下文**，而不是把它直接解释为成功或失败。
